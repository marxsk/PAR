PAR is a question answering system created by Matej Stepanek at Faculty of Informatics at Masaryk University. This system is a part of the master's thesis "Automatické zodpovídání dotazů a internetové vyhledávače" (Question Answering and Web Search Engines), where you can find more information.
January 2015


How does PAR work?
Answerer
  • You give a question.
  • PAR processes the question and sends it to the web search engine (Seznam or Google).
  • The search engine returns links to relevant documents.
  • PAR identifies keywords in question.
  • For every link PAR does following:
    • gets a content of the web page,
    • finds a paragraph that contains the most keywords.
  • From these paragraphs PAR returns the best one.

Testing

  • When testing, PAR finds the best paragraph and then searches in it for words from the answer.
  • Paragraph is considered as a correct result if it contains every word from the answer or if
    the answer is included in the paragraph and in the title of it's web page together.

General

  • When searching for matches, all words are converted to lemmata.
  • Both inputs and outputs are in utf-8 encoding.
  • Google results are gained by Custom Search JSON API. (There is a limit - 100 queries per day.)
  • Seznam results are gained by Seznam Search API without any restriction.
  • justext by Jan Pomikálek is used to gain a page content without boilerplate.
  • unitok.py by Jan Pomikálek is used for tokenization.
  • majka by Pavel Šmerk is used for lemmatization.
